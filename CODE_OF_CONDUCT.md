# Code of Conduct

This project exists to improve consumer transparency and reduce harm in
digital systems. To make that possible, we expect all participants to
behave respectfully and constructively.

---

## 1. Expected behavior

Participants in issues, pull requests, and discussions should:

- Use clear, respectful language
- Assume good faith while still being able to question ideas
- Focus critiques on arguments and proposals, not people
- Keep discussions relevant to CTGS and related governance topics

---

## 2. Unacceptable behavior

The following are not acceptable in this project:

- Personal attacks, insults, or harassment
- Hate speech or discriminatory language
- Deliberate misinformation about CTGS or its purpose
- Persistent off-topic or bad-faith engagement

---

## 3. Scope

This Code of Conduct applies to:

- GitHub issues and pull requests
- Discussions related to this repository
- Any shared communication channels used by maintainers for CTGS work

---

## 4. Enforcement

Project maintainers may:

- Request a change in tone or behavior
- Close issues or pull requests that violate this Code
- Block or restrict users who repeatedly or severely violate expectations

These actions are taken at the maintainersâ€™ discretion, with the goal of
keeping the project focused and safe.

---

## 5. Reporting

If you experience or see behavior that violates this Code of Conduct:

- Open an Issue if you feel comfortable doing so; or
- Contact a maintainer directly if such a channel is provided.

Please provide enough detail for maintainers to understand what happened
and where.

---

This Code of Conduct is intentionally concise. It is meant to support
productive collaboration on CTGS and related tooling.
