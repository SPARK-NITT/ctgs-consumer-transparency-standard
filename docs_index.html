What CTGS is

The Consumer Transparency Governance Standard (CTGS) defines how consumer-facing systems must behave when things go wrong. It focuses on recursive harm patterns such as:

  • repeating setup / reset loops that never resolve (failure cycles)
  • compounding “fix” costs that exceed the value of the product
  • hidden ecosystem locks and incompatibilities
  • float capture during returns, upgrades, and disputes
  • AI recommenders that steer users into known failure patterns

This repository contains the CTGS v1.0 standard, implementation notes, licensing, and integrity proofs (hashes and notarization).

Key files in this repository

  • Standard_v1.0.md — formal CTGS v1.0 specification.
  • Implementation_Notes_v1.0.md — non-normative guidance and examples.
  • LICENSE — MIT License for code and tooling.
  • LICENSE-CC-BY-4.0.md — CC BY 4.0 for the standard text.
  • HASHES.md, HASHES.md.ots, NOTARIZATION.md — integrity and notarization chain.

Additional folders such as docs/, schemas/, or ctgs/ can be added over time as the project grows.
